# Iteration 7 — Revisions

## 1. User-definable episode dates & title

**Dependency added:** `@react-native-community/datetimepicker` (installed, in `app.config.ts` plugins)

### Screens to update
- `src/screens/main/NewEpisodeScreen.tsx`
- `src/screens/onboarding/EpisodeSetupScreen.tsx`

### Changes
- **Start date**: editable via `DateTimePicker` (mode `date`). Defaults to today, but user can pick any past or future date (supports retroactive logging).
- **End date**: editable via `DateTimePicker`. Defaults to end of current quarter (`getQuarterEndDate`). Validation: must be after start date.
- **Title**: editable `TextInput`. Default generated by new helper `getDefaultEpisodeTitle(startDate)`:
  - Queries existing episodes that overlap the same quarter as `startDate`
  - If none exist → `"Q1 2026"`
  - If N already exist → `"Q1 2026 e{N+1}"` (e.g. `"Q1 2026 e2"`, `"Q1 2026 e3"`)
- Replace the static `dateDisplay` views with touchable date fields that open `DateTimePicker` on press.
- On Android, `DateTimePicker` shows as a modal dialog (default behavior). On iOS, show inline or as modal — use platform-appropriate UX.

### Files also touched
- `src/utils/dates.ts` — add `getDefaultEpisodeTitle(startDate, existingEpisodes)` helper
- `src/db/repositories/episodes.ts` — may need a query to count episodes in a quarter range for title generation

## 4. Report export: proper filename + copy-to-clipboard

### Screens to update
- `src/screens/main/ReportsScreen.tsx`
- `src/screens/main/ReportSuccessScreen.tsx`

### Changes — file-based sharing
- When sharing a report, write JSON to a temp file via `expo-file-system` (`FileSystem.documentDirectory`), then share the file URI instead of raw text.
- **Filename format**: `gordon-report-{episode-title}-{YYYY-MM-DD}.json`
  - Episode title slugified (lowercase, spaces → hyphens, strip special chars)
  - Date is the export date
  - Example: `gordon-report-q1-2026-2026-02-14.json`

### Changes — copy JSON button
- `ReportSuccessScreen.tsx`: replace the stub `Alert` with actual clipboard copy using React Native's `Clipboard` API (available via `react-native`'s built-in module, no new dependency).
- Show a brief confirmation (e.g. `Alert.alert('Copied!', 'Report JSON copied to clipboard.')`) after successful copy.

### Changes — export tracking
- `ReportSuccessScreen.tsx`: call `markReportExported()` after sharing (currently only `ReportsScreen` does this).
